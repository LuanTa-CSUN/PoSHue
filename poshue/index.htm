What's this?

Philips have now made available an API which allows you to control your Philips Hue lights remotely, using another application, such as a website, or my PowerShell module (class) that is not owned and operated by Philips themselves.

If you're here, then it's likely you've arrived from use of my PowerShell Module for controlling Philips Hue luminaires and want to be able to access and control those luminaires from a remote location that is not on the same Local Area Network (LAN) as your Philips Hue Bridge.

Why do I need to do this?

In order to control your lights, groups and sensors remotely, you must send commands to your Hue Bridge via the Philips Hue Remote API. To do that, you must tell Philips that you trust your copy of the PosHue module to act on your behalf and you do that by creating an Access Token and providing it when using the module.

Do I need to do this?

If you intend to use the module from a computer on the same network as your Philips Hue Bridge, then no. If however you want to control your lights from an Azure Automation Runbook or a Linux server running in Amazon Web Services, then yes.

What do I need to do?

It's simple enough. Click the button below then log in to your Philips Hue Account . You'll be asked to grant permission to the PoSHue application. The process of granting permission generates the access token (which looks like a random set of letters and numbers) which you can then use in the module.

What are you going to do with this token?

Absolutely nothing at all. I don't see it, record it, store it, read it, use it or do anything with it whatsoever. As soon as you close the page with your token shown, that's it - gone. This is your token and your responsibility to keep secret.
